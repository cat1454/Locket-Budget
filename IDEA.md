# Locket Budget - Product Idea

## 1. Tinh than du an

Locket Budget la ung dung quan ly chi tieu ca nhan theo phong cach hien dai, gan gui va nhieu cam xuc nhu Locket. Thay vi chi nhap so lieu kho khan, nguoi dung ghi lai moi khoan chi bang anh, so tien va mot ghi chu ngan. Moi lan chi tieu tro thanh mot manh nho trong nhat ky doi song.

Gia tri cot loi cua du an khong chi la "quan ly tien" ma la "nho minh da tieu gi, vao luc nao, trong boi canh nao".

## 2. Bai toan can giai quyet

Nhieu nguoi, dac biet la sinh vien va nguoi tre moi di lam, gap cac van de sau:

- Kho duy tri thoi quen ghi chep chi tieu hang ngay.
- Ung dung tai chinh truyen thong thuong kho khan, nhieu bieu mau, thieu cam hung.
- Khi chi nhin vao con so, nguoi dung kho nho minh da tieu vao viec gi.
- Khong de nhan ra thoi quen chi tieu xau de dieu chinh.
- Thieu dong luc tiet kiem va quay lai su dung app lau dai.

## 3. Giai phap de xuat

Ung dung cho phep nguoi dung:

- Chup anh hoac chon anh cho moi khoan chi.
- Nhap so tien, ghi chu ngan, chon danh muc.
- Luu lai thoi gian chi tieu.
- Xem lich su theo dang timeline hinh anh.
- Xem tong chi tieu theo ngay, tuan, thang.
- Chinh sua va xoa khoan chi.

Ve sau, he thong co the mo rong them mood spending, widget, challenge tiet kiem, badge thanh tich va goi y thong minh.
Sau cac lop mo rong do, san pham co the them chia tien nhom va mot lop ket noi xa hoi nhe de nguoi dung chia se nhung `moment` da chon voi ban be.

## 4. Dinh vi san pham

Locket Budget khong canh tranh bang bo feature ke toan phuc tap. Dinh vi cua san pham la:

- Nhanh hon app ghi chep chi tieu thong thuong.
- Truc quan hon bang hinh anh thay vi chi co so lieu.
- Gan gui hon nho giong mot nhat ky doi song.
- De duy tri hon nho tuong tac nhe, dep va co cam xuc.

San pham cung khong nen tro thanh mot mang xa hoi anh thong thuong. Neu them ket ban va chia se, huong dung phai la `social-lite`, nghia la chi bo sung ket noi nhe xoay quanh cac moment chi tieu va mot vai khoanh khac doi song duoc chon loc.

## 5. Doi tuong nguoi dung

### 5.1. Nhom chinh

- Sinh vien muon quan ly tien tieu vat hang ngay.
- Nguoi tre moi di lam muon kiem soat chi tieu ca nhan.
- Nguoi thich chup anh, luu ky niem doi song va muon gan viec chi tieu voi boi canh that.

### 5.2. Nhom phu

- Nguoi muon bat dau quan ly tai chinh nhung ngai app qua "nghiep vu".
- Nguoi muon xay dung thoi quen tiet kiem theo cach nhe nhang, khong ap luc.

## 6. Tam nhin trai nghiem

Ung dung can theo cac nguyen tac sau:

- Photo-first: anh la thanh phan trung tam cua moi khoan chi.
- Mobile-first: thao tac nhanh, it buoc, uu tien dung bang mot tay.
- Emotional UX: giao dien than thien, hien dai, tao cam giac gan gui.
- Low friction: nguoi dung co the tao mot khoan chi trong vai giay.
- Personal diary: timeline phai tao cam giac "day la cau chuyen chi tieu cua toi".
- Social-lite, not social-first: neu co ket ban va chia se, Home van phai la noi ghi chi tieu nhanh, khong bien thanh feed.

## 14. Nguyen tac phat trien

De du an khong bi vo pham vi, can bam sat cac nguyen tac sau:

1. Lam xong MVP truoc khi them tinh nang thu vi.
2. Moi giai doan phai co dau ra chay duoc.
3. Giao dien dep nhung khong duoc lam cham thao tac chinh.
4. Du lieu va luong nguoi dung phai don gian, ro rang, de kiem thu.
5. Moi tinh nang mo rong chi duoc them khi core da on dinh.
6. Neu them ket ban va chia se, phai dam bao no tang gia tri cho expense diary thay vi lam loang dinh vi san pham.

## 15. Dac ta du lieu cho Social Lite

Phan nay dung de chot data model toi thieu cho `Phase 5`, giup viec code backend, schema database va API khong bi mo ho.

### 15.1. Entity `Friendship`

Muc dich:

- Luu quan he ket noi giua 2 user.
- Quan ly trang thai loi moi ket ban va cac hanh vi nhu chap nhan, tu choi, huy ket ban, chan.

Field de xuat:

- `id`: string, dinh danh duy nhat.
- `requesterId`: string, user gui loi moi.
- `addresseeId`: string, user nhan loi moi.
- `pairKey`: string, khoa duy nhat duoc tao tu 2 user id sau khi sap xep tang dan, vi du `userA:userB`.
- `status`: enum `pending | accepted | declined | removed | blocked`.
- `createdAt`: datetime, thoi diem tao quan he hoac gui loi moi.
- `respondedAt`: datetime nullable, thoi diem chap nhan hoac tu choi.
- `updatedAt`: datetime, thoi diem cap nhat gan nhat.

Rang buoc:

- `requesterId` khac `addresseeId`.
- `pairKey` la duy nhat tren toan he thong, tranh 2 ban ghi friendship cho cung 1 cap user.
- Neu da co `status = blocked`, khong duoc tao moi friendship khac cho cung `pairKey`.
- V1 nen cap nhat tren cung 1 ban ghi friendship thay vi tao lich su nhieu ban ghi.

Quan he:

- Mot `User` co the xuat hien nhieu lan trong `Friendship` voi vai tro requester hoac addressee.

Chi muc nen co:

- Unique index: `pairKey`.
- Index: `requesterId`, `addresseeId`, `status`.

### 15.2. Entity `ShareMoment`

Muc dich:

- Luu mot `moment` ma user chon de chia se voi ban be.
- Moment co the den tu mot khoan chi (`expense moment`) hoac mot anh thong thuong (`normal photo moment`).

Field de xuat:

- `id`: string, dinh danh duy nhat.
- `ownerUserId`: string, user tao moment.
- `momentType`: enum `expense | normal`.
- `expenseId`: string nullable, bat buoc khi `momentType = expense`.
- `imageUri`: string, duong dan anh goc hoac storage key.
- `thumbnailUri`: string nullable, anh thu nho de load feed nhanh.
- `caption`: string nullable, ghi chu ngan do user nhap khi share.
- `privacyScope`: enum `private | friends_only`.
- `shareAmountPreview`: boolean, co cho hien preview so tien hay khong.
- `amountSnapshot`: number nullable, gia tri duoc dong bang tai thoi diem share.
- `categorySnapshot`: string nullable, category duoc dong bang tai thoi diem share.
- `noteSnapshot`: string nullable, note duoc dong bang tai thoi diem share.
- `createdAt`: datetime, thoi diem tao record.
- `sharedAt`: datetime, thoi diem moment duoc cong khai cho ban be.
- `updatedAt`: datetime, thoi diem cap nhat gan nhat.
- `deletedAt`: datetime nullable, phuc vu soft delete.

Rang buoc:

- `ownerUserId` la bat buoc.
- Neu `momentType = expense` thi `expenseId` phai co.
- Neu `momentType = normal` thi `expenseId` phai la `null`.
- `imageUri` la bat buoc.
- `privacyScope` chi nen co `private` va `friends_only` trong v1 de tranh phuc tap hoa data model.
- V1 nen gioi han moi `Expense` chi tao toi da 1 `ShareMoment` dang active; neu muon share lai thi cap nhat ban ghi cu.

Ghi chu thiet ke:

- `ShareMoment` nen luu snapshot (`amountSnapshot`, `categorySnapshot`, `noteSnapshot`) thay vi doc truc tiep tu `Expense` moi lan render.
- Cach nay giu feed on dinh neu user sua hoac xoa expense goc sau khi da share.
- Neu sau nay can share cho mot nhom ban duoc chon thay vi tat ca ban be, nen them bang phu `ShareMomentAudience` thay vi luu mang user id trong `ShareMoment`.

Quan he:

- Mot `User` co nhieu `ShareMoment`.
- Mot `Expense` co 0 hoac 1 `ShareMoment` active trong v1.
- Mot `ShareMoment` co nhieu `MomentReaction`.

Chi muc nen co:

- Index: `ownerUserId`, `privacyScope`, `sharedAt`.
- Index: `expenseId`.
- Index phuc vu feed: `(ownerUserId, deletedAt)` hoac `(sharedAt, deletedAt)`.

### 15.3. Entity `MomentReaction`

Muc dich:

- Luu reaction nhe cua ban be tren mot moment.
- V1 chi ho tro icon reaction, khong them comment text de tranh bien app thanh mang xa hoi day du.

Field de xuat:

- `id`: string, dinh danh duy nhat.
- `momentId`: string, moment duoc react.
- `userId`: string, user tha reaction.
- `reactionType`: enum `heart | fire | wow | cheer`.
- `createdAt`: datetime.
- `updatedAt`: datetime.

Rang buoc:

- `momentId` va `userId` la bat buoc.
- User chi duoc react vao moment ma minh co quyen xem.
- V1 nen gioi han moi user toi da 1 reaction tren 1 moment; neu doi icon thi update ban ghi cu.
- Khong can them text comment trong `MomentReaction`; neu sau nay can comment thi tach bang rieng.

Quan he:

- Mot `ShareMoment` co nhieu `MomentReaction`.
- Mot `User` co nhieu `MomentReaction`.

Chi muc nen co:

- Unique index: `(momentId, userId)`.
- Index: `momentId`, `userId`.

### 15.4. Luong quan he tong quat

- `User` 1-n `ShareMoment`
- `User` 1-n `MomentReaction`
- `ShareMoment` 1-n `MomentReaction`
- `User` n-n `User` thong qua `Friendship`
- `Expense` 0..1-1 `ShareMoment` trong pham vi v1 social-lite

### 15.5. Quy uoc nghiep vu de tranh mo ho khi code

- Home khong doc du lieu tu `ShareMoment`; Home chi phuc vu ghi nhanh chi tieu.
- Feed ban be doc tu `ShareMoment`, khong doc truc tiep tu `Expense`.
- Quyen xem `friends_only` duoc xac dinh bang `Friendship.status = accepted`.
- Neu `Friendship.status` chuyen sang `removed` hoac `blocked`, moment cu `friends_only` khong con hien cho user do nua.
- `shareAmountPreview = false` la mac dinh an toan cho expense moment.
- `normal photo moment` la tuy chon va khong duoc yeu cau trong core flow.

### 15.6. ERD text cho Social Lite

```text
User
  id PK
  displayName
  email
  ...

Expense
  id PK
  userId FK -> User.id
  categoryId
  amount
  note
  imageUri
  ...

Friendship
  id PK
  requesterId FK -> User.id
  addresseeId FK -> User.id
  pairKey UNIQUE
  status
  createdAt
  respondedAt
  updatedAt

ShareMoment
  id PK
  ownerUserId FK -> User.id
  expenseId FK -> Expense.id NULL
  momentType
  imageUri
  thumbnailUri
  caption
  privacyScope
  shareAmountPreview
  amountSnapshot
  categorySnapshot
  noteSnapshot
  createdAt
  sharedAt
  updatedAt
  deletedAt

MomentReaction
  id PK
  momentId FK -> ShareMoment.id
  userId FK -> User.id
  reactionType
  createdAt
  updatedAt

Relationships
  User 1 --- n Friendship (as requester)
  User 1 --- n Friendship (as addressee)
  User 1 --- n ShareMoment
  Expense 1 --- 0..1 ShareMoment
  ShareMoment 1 --- n MomentReaction
  User 1 --- n MomentReaction
```

Doc cach hieu:

- `Friendship` la bang trung gian giua `User` va `User`, nhung co them `status`.
- `ShareMoment` co the gan voi `Expense` hoac khong, tuy `momentType`.
- `MomentReaction` la bang con cua `ShareMoment`, gioi han 1 user 1 reaction cho moi moment.

### 15.7. API contract v1 de xuat

Nguyen tac:

- Tat ca API deu yeu cau user da dang nhap.
- Response v1 nen giu shape don gian, uu tien de mobile de map vao UI.
- Social feed chi doc `ShareMoment`, khong doc truc tiep `Expense`.

#### A. Send friend request

`POST /api/v1/friendships/requests`

Request body:

```json
{
  "addresseeId": "user_002"
}
```

Response `201 Created`:

```json
{
  "data": {
    "id": "friendship_001",
    "requesterId": "user_001",
    "addresseeId": "user_002",
    "pairKey": "user_001:user_002",
    "status": "pending",
    "createdAt": "2026-03-28T10:00:00.000Z",
    "respondedAt": null,
    "updatedAt": "2026-03-28T10:00:00.000Z"
  }
}
```

Loi co the co:

- `409` neu da ton tai friendship voi `pairKey` nay.
- `400` neu gui loi moi cho chinh minh.
- `403` neu mot ben da block ben con lai.

#### B. Accept or decline friend request

`POST /api/v1/friendships/{friendshipId}/respond`

Request body:

```json
{
  "action": "accept"
}
```

`action` hop le: `accept | decline | block`

Response `200 OK`:

```json
{
  "data": {
    "id": "friendship_001",
    "status": "accepted",
    "respondedAt": "2026-03-28T10:05:00.000Z",
    "updatedAt": "2026-03-28T10:05:00.000Z"
  }
}
```

#### C. Remove friend

`DELETE /api/v1/friendships/{friendshipId}`

Response `200 OK`:

```json
{
  "data": {
    "id": "friendship_001",
    "status": "removed"
  }
}
```

`POST /api/v1/share-moments/{momentId}/reactions`

Request body:

```json
{
  "reactionType": "heart"
}
```

Response `200 OK`:

```json
{
  "data": {
    "id": "reaction_001",
    "momentId": "moment_001",
```
